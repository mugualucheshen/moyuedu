## 四、发现并修复的 bug

### Bug 1：导出功能 JS 语法错误 🔴 P0

- **症状**：导出书架时 `apiKey: ***` 是非法 JS，会让整个脚本启动失败
- **触发**：用户点击"导出书架"
- **修复**：改为解构过滤 API Key
  ```js
  settings: (() => { const { apiKey, ...rest } = state.settings; return rest; })()
  ```
- **状态**：已修复 ✅

### Bug 2：章节正则大文本下卡死 + 贪婪吞标题 🔴 P0

- **症状 A**：原始 `^\\s*pattern.*$` + `m` flag 在 10MB 文本中第二次匹配失败（offset 错位）
- **症状 B**：`[^\n]*` 贪婪匹配，会从"第1章"一路吞到"第2章"，导致第二个章节找不到
- **触发**：TXT 章节之间有 5MB+ 无换行内容
- **修复**：
  - 锚点改为 `(?:^|\n)`（非捕获，0 宽）
  - 标题长度限制为 `[^\\n]{0,80}`
  - 大文本无章节时降级为"每 3000 字一个虚拟段"
- **状态**：已修复 ✅
- **性能**：10MB / 1000 章节解析 **3ms**（超预期）

### Bug 3：审计发现的"非真 bug"（T10 误报）

- **症状**：测试期望"无换行的 10MB 长字符串"也能正确分章
- **结论**：这是病态用例（真实 TXT 不可能无换行），不修
- **处理**：测试改为"1000 章节 × 10000 字"的真实场景

## 六、部署指引（怎么传到 iPhone）

### 方式 A：双击 HTML（最简单）

1. 把 `index.html` 通过微信"文件传输助手" / 邮件 / iCloud Drive 传到 iPhone
2. 在 iPhone 上用 **"文件" App** 找到文件
3. 长按 → "共享" → "用 Safari 打开"
4. 首次进入会弹窗让填小米 API Key（可点"稍后配置"跳过）
5. Safari 底部分享 → "添加到主屏幕" → 像 App 一样用 ✅

### 方式 B：自托管（推荐，可当 PWA）

1. 老板电脑起服务：
   ```bash
   cd 产出物
   python -m http.server 8765
   ```
2. 查看电脑 IP（局域网）：
   ```bash
   ipconfig  # Windows
   ```
   假设得到 `192.168.1.100`
3. iPhone Safari 访问 `http://192.168.1.100:8765/index.html`
4. Safari → 分享 → "添加到主屏幕"
5. 桌面图标，点击像原生 App ✅

### 方式 C：公网托管（可选，需要时再做）

- **Surge.sh**（免费）：`npm i -g surge && surge .`
- **Vercel**（免费）：`npx vercel`
- **GitHub Pages**：推到仓库开 Pages
